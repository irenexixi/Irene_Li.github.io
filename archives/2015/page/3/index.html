1、Directive中的compile & link

先从directive是如何在angular下生效的说起吧~

编译三阶段：

1. 标准浏览器API转化

将html转化成dom，所以自定义的html标签必须符合html的格式

2. Angular compile

搜索匹配directive，按照priority(优先)排序，并执行directive上的compile方法

3. Angular link

执行directive上的link方法，进行scope绑定及事件绑定

为什么编译的过程要分成compile和link?

简单的说就是为了解决性能问题，特别是那种model变化会影响dom结构变化的，而变化的结构还会有新的scope绑定及事件绑定，比如ng-repeat

compile和link的形式

compile

function compile(element, attrs, transclude) { … }
在compile阶段要执行的函数，返回的function就是link时要执行的function
常用参数为element和attrs，分别是dom元素和元素上的属性们，其它的以后细说
较少使用，因为大部分directive是处理dom元素的行为绑定，而不是改变它们

link

function link(scope, element, attrs, controller) { … }
在link阶段要执行的函数，这个属性只有当compile属性没有设置时才生效
常用参数为scope，element和attrs，分别是当前元素所在的scope，dom元素和元素上的属性们，其它的以后细说
directive基本上都会有此函数，可以注册事件，并与scope相绑

compile和link的使用时机

compile

想在dom渲染前对它进行变形，并且不需要scope参数
想在所有相同directive里共享某些方法，这时应该定义在compile里，性能会比较好
返回值就是link的function，这时就是共同使用的时候

link

对特定的元素注册事件
需要用到scope参数来实现dom元素的一些行为

2、$scope.$apply()

在写AngularJs代码时，通常会遇到$scope.$apply()方法，表面上看起来是在模型有更新时执行，我确实是这样做的,实际上是有很深的意义在里面的。
在网上看到一篇文章，不知道怎样用中文来表达，还是附上地址吧$scope.$apply()

$apply and $digest

That step that checks to see if any binding values have changed actually has a method, $scope.$digest(). That’s actually where the magic happens, but we almost never call it directly, instead we use $scope.$apply() which will call $scope.$digest() for you.

$scope.$apply() takes a function or an Angular expression string, and executes it, then calls $scope.$digest() to update any bindings or watchers.

$scope.$apply() vs $scope.$apply(fn)

Sometimes I see examples where data is updated, and then $scope.$apply() is called with no arguments. This achieves the desired result, but misses some opportunities.

If your code isn’t wrapped in a function passed to $apply, and it throws an error, that error is thrown outside of AngularJS, which means any error handling being used in your application is going to miss it. $apply not only runs your code, but it runs it in a try/catch so your error is always caught, and the $digest call is in a finally clause, meaning it will run regardless of an error being thrown. That’s pretty nice.

Hopefully now you understand what $apply is and when to use it. If you only use what AngularJS provides you, you shouldn’t need to use it often. But if you begin writing directives where you are observing DOM elements directly, it is going to become necessary.

使用$scope.$apply(fn)更好。

html5-form-attr

1、表单内元素的form属性

HTML4中，表单的从属元素必须写在表单内部，而HTML5中，可以书写在页面上任何地方，然后为该元素指定一个form属性，属性值为该表单的id值，这样就可以声明该元素从属于指定表单了。

兼容性
Safari 5+， Google Chrome 10+， Firefox 4+， Opera 10+

2、表单内元素的formaction属性

HTML4中，表单内的所有元素只能通过表单的action属性被统一提交到另一个页面，而HTML5中可以给提交按钮增加不同的formaction属性，使单击不同的按钮时可以提交到不同的页面。

兼容性

Safari 5+， Google Chrome 10+， Firefox 4+， Opera 10.50+，IE 10

3、表单内元素的formmethod属性

HTML4中，表单内只有一个method属性来统一指定提交方法，HTML5中可以使用formmethod属性对每个表单元素分别指定不同的提交方法。

兼容性

Safari 5+， Google Chrome 10+， Firefox 4+， Opera 10.50+，IE 10

4、表单内元素的formenctype属性

HTML4中，有一个enctype属性用于指定在表单发送到服务器之前应该对表单内数据进行编码，HTML5中可以使用formmethod属性表单元素分别指定不同的编码方式。

编码方式有以下3种：

  • multipart/form-data

    不对字符串编码，对使用包含文件上传控件的表单时必须使用该值。

  • text/plain

    对表单数据中的空格被转换成“+”， 不对表单中的特殊字符进行编码。

  • application/x-www-form-urlencoded

    enctype属性的默认值，在发送前编码所有字符。当提交方法为get时，会把数据转换成字符串（？name1=value1&name2=value2···），然后把这个字符串添加到url地址之后，成为新的目标url地址。

兼容性

Safari 5+， Google Chrome 10+， Firefox 4+， Opera 10.50+，IE 10

5、表单内元素的formtarget属性

HTML4中，target属性用于指定在何处打开表单提交后所需要加载的页面，HTML5中可以使用formtarget属性可以对多个提交按钮分别指定formtarget值。

formtarget属性值有以下3种：

  • _blank: 在新的窗口中打开。

  • _self： 默认值。在相同的框架中打开。

  • _parent： 在当前框架的父框架中打开。

  • _top： 在当前浏览器窗口中打开。

  • framename： 在指定的框架中打开。

兼容性

Safari 5+， Google Chrome 10+， Firefox 4+， Opera 10.50+，IE 10

6、表单内元素的autofocus属性

HTML5中新增属性，为文本框，选择框，按钮控件加上autofocus属性时，页面在打开时该控件会自动获得光标。目前要实现这个功能需要使用js，例如：$input.focus()。

一个页面中只能有一个控件具有autofocus属性，建议一个页面以一个控件为主要目的时，才使用autofucs属性，例如搜索页面的搜索框。

兼容性

Safari 5+， Google Chrome 6+， Firefox 4+， Opera 10+，IE 10

7、表单内元素的required属性

HTML5中新增属性，可以应用在大多数输入元素上（除了隐藏元素，图片元素按钮上）,提交时，如果元素内容为空，则不允许提交，同时显示用户提示信息。

兼容性

Safari 5+， Google Chrome 6+， Firefox 4+， Opera 10+，IE 10

8、表单内元素的labels属性

HTML5中新增属性，对可以使用标签元素（label元素）的表单元素input（除了type为hidden的元素）、textarea、select、button、meter、output、progress、keygen等定义一个labels属性，属性值为一个Nodelist对象，代表该元素所绑定的标签元素的所构成的集合。

兼容性

Safari 5+， Google Chrome 6+， Opera 10+

9、表单内元素的control属性

HTML5中新增属性，在标签label内部放置一个表单元素，可以通过control属性访问这个表单元素，即把control属性的值设置成元素元素的value。

一个页面中只能有一个控件具有autofocus属性，建议一个页面以一个控件为主要目的时，才使用autofucs属性，例如搜索页面的搜索框。

兼容性

Safari 5+， Google Chrome 18+，Firefox 10+, Opera 12+

10、表单内元素的placeholder属性

HTML5中新增属性，可以设置表单元素input、textarea处于未输入状态时的提示文字。

可以通过css设置提示文字的样式

兼容性

Safari 5+， Google Chrome 6+，Firefox 4+, Opera 11+、IE 10+

11、表单内元素的list属性

HTML5中新增属性，该属性的值为某个datalist元素的id。当输入框获取焦点并输入内容时若有匹配则显示datalist的内容

兼容性

Google Chrome 20+，Firefox 10+, Opera 10+、IE 10+

12、表单内元素的autocomplete属性

HTML5中新增属性，有三个值on/off/“ “， on时可以显示指定候补输入的数据列表。当不指定值时，使用浏览器的默认值（取决于各浏览器）。

兼容性

Safari 6.0+，Google Chrome 14+，Firefox 4+, Opera 10+、IE 10+

13、表单内元素的pattern属性

HTML5中新增属性，对input元素使用pattern属性，属性值设置为某个格式正则表达式，表单提交时会检验输入元素内容是否符合给定格式，如果不符合则不允许提交，并且会弹出相应的提示信息。

兼容性

Safari 5.0+，Google Chrome 6+，Firefox 4+, Opera 10+、IE 10+

14、表单内元素的selectionDirection属性

HTML5中新增属性，针对input和textarea元素，可以获取选取元文本框中文字的方向。

默认值和正向选取值为”forward”, 反向选取值为”backward”.

兼容性

全局属性: 指可以对任何元素都使用的属性

1. contentEditable属性

主要功能是允许用户编辑元素中的内容，所以该元素必须是可以获得鼠标焦点的元素
点击鼠标后会向用户提供一个插入符号，提示该元素中的内容是允许编辑的。

contentEditable属性是一个布尔值属性，true/false/inherit。
想要保存编辑后的内容，只能把该元素的innerHtml发送到服务端，目前还没有特定的API来保存编辑后的元素的内容。

元素还存在一个属性，isContentEditable,当元素可编辑时，该属性为true，当元素不可编辑时，为false。

使用contenteditable属性时需要注意，他的实现原理类似于编辑器，在复制内容到指定元素时，会连同内容的style也会复制，可能达不到开发人员预期想要的结果。

2. designMode属性

designMode属性用来指定整个页面是否可编辑，当页面可编辑时，页面中所有支持contentEditable属性的元素都变成可编辑状态。只能在js脚本里被编辑修改。
designMode属性有两个值on/off。on，页面可编辑，off页面不可编辑

兼容性

  • ie8: 出于安全考虑，不允许使用
  • Chrome 3和Safari: 使用内嵌frame的方式，该内嵌frame是可编辑的
  • ie9/Firefox/Opera/Chrome3+: 允许

hidden属性

功能是通知浏览器不渲染该元素，使该元素处于不可见状态，类似于input元素中的hidden元素

页面加载后可以使用js脚本取消该值，取消后该元素变成可见状态，显示元素的内容。

hidden属性的值：true/false，true元素不可见，false，元素可见。

3. spellcheck属性

功能是对用户输入的文本内容进行拼写和语法检查。 针对input、textarea元素有效。

spellcheck属性的值：true/false，他在书写时有个特殊的地方，必须明确声明属性值是true或false

如果元素的readonly属性或disable属性为true时，不会执行检查

兼容性

除了IE，其他浏览器都支持

Js

js-event

概述

JavaScript和HTML的交互是通过事件实现的。JavaScript采用异步事件驱动编程模型，当文档、浏览器、元素或与之相关对象发生特定事情时，浏览器会产生事件。如果JavaScript关注特定类型事件，那么它可以注册当这类事件发生时要调用的句柄。

事件流

事件流描述的是从页面中接收事件的顺序，比如有两个嵌套的div，点击了内层的div，这时候是内层的div先出发click事件还是外层先触发？目前主要有三种模型

  1. IE的事件冒泡：事件开始时由最具体的元素接收，然后逐级向上传播到较为不具体的元素
  2. Netscape的事件捕获：不太具体的节点更早接收事件，而最具体的元素最后接收事件，和事件冒泡相反
  3. DOM事件流：DOM2级事件规定事件流包括三个阶段，事件捕获阶段，处于目标阶段，事件冒泡阶段，首先发生的是事件捕获，为截取事件提供机会，然后是实际目标接收事件，最后是冒泡句阶段。

Opera、Firefox、Chrome、Safari都支持DOM事件流，IE不支持事件流，只支持事件冒泡

参考：Javascript与Html交互-事件
